🧹 Finding Missing Values

Unit 6 — Real-world data is messy. Learn to find and fix every type of data quality problem.

Why Data is Missing

Missing data happens for many reasons: form fields left blank, system errors, data not collected yet. Pandas represents missing values as NaN (Not a Number).

python
import pandas as pd import numpy as np df = pd.DataFrame({ "Name": ["Ahmed","Sara",None,"Layla","Karim"], "Sales": [45000, np.nan, 38000, np.nan, 55000], "Region": ["Cairo","Alex","Cairo",None,"Giza"], "Years": [3, 5, np.nan, 7, 2], }) # Overview of missing data print("isnull():") print(df.isnull()) print("\nMissing per column:") print(df.isnull().sum()) print("\n% missing:") print((df.isnull().sum() / len(df) * 100).round(1))
Output
Missing per column: Name 1 Sales 2 Region 1 Years 1 dtype: int64 % missing: Name 20.0 Sales 40.0 Region 20.0 Years 20.0 dtype: float64

Visualising Missing Data

python
# Rows with ANY missing value print(f"Rows with missing: {df.isnull().any(axis=1).sum()}") # Show only rows that have missing values print(df[df.isnull().any(axis=1)]) # Heatmap of missing (use missingno library for visuals) # import missingno as msno # msno.matrix(df)
💡 Tip: Always check missing values FIRST before any analysis. df.info() gives a quick count of non-null values per column.

🧹 Filling & Dropping Missing Data

Filling Missing Values

python
import pandas as pd, numpy as np df = pd.DataFrame({ "Sales": [45000,np.nan,38000,np.nan,55000,42000], "Region": ["Cairo",None,"Alex","Cairo",None,"Giza"], "Month": [1,2,3,4,5,6], }) # Fill with a fixed value df2 = df.copy() df2["Region"].fillna("Unknown", inplace=True) # Fill numeric with mean / median df2["Sales"].fillna(df["Sales"].mean(), inplace=True) # Forward fill (use previous value) df3 = df.copy() df3["Sales"].fillna(method="ffill", inplace=True) # Backward fill df3["Region"].fillna(method="bfill", inplace=True) # Fill with group mean df["Sales"] = df.groupby("Region")["Sales"].transform( lambda x: x.fillna(x.mean()) ) print(df)
Output
Sales Region Month 0 45000.0 Cairo 1 1 38000.0 Alex 2 2 38000.0 Alex 3 3 45000.0 Cairo 4 4 55000.0 Giza 5 5 42000.0 Giza 6

Dropping Missing Values

python
# Drop rows with ANY missing df.dropna(inplace=True) # Drop rows where SPECIFIC column is missing df.dropna(subset=["Sales"], inplace=True) # Drop columns with too many missing (>50%) threshold = len(df) * 0.5 df.dropna(thresh=threshold, axis=1, inplace=True) # Drop if ALL values in row are NaN df.dropna(how="all", inplace=True)
💡 Tip: Prefer filling over dropping when possible — dropping rows loses data. Use group-based filling for the most accurate imputation.

🧹 Detecting Outliers

What is an Outlier?

An outlier is a value unusually far from the rest. It might be a data entry error, a system glitch, or a genuine extreme event — you need to investigate before removing.

Z-Score Method

python
import pandas as pd, numpy as np sales = pd.Series([45000,52000,48000,61000,1200000,58000,31000,55000]) # Z-score: how many standard deviations from the mean z_scores = (sales - sales.mean()) / sales.std() print("Z-scores:") print(z_scores.round(2)) print(f"\nOutliers (|z| > 3): {sales[abs(z_scores) > 3].tolist()}")
Output
Z-scores: 0 -0.64 1 -0.47 2 -0.56 3 -0.26 4 6.64 5 -0.38 6 -0.93 7 -0.41 dtype: float64 Outliers (|z| > 3): [1200000]

IQR Method (More Robust)

python
import pandas as pd, numpy as np sales = pd.Series([45000,52000,48000,61000,1200000,58000,31000,55000]) Q1 = sales.quantile(0.25) Q3 = sales.quantile(0.75) IQR = Q3 - Q1 lower_fence = Q1 - 1.5 * IQR upper_fence = Q3 + 1.5 * IQR outliers = sales[(sales < lower_fence) | (sales > upper_fence)] clean = sales[(sales >= lower_fence) & (sales <= upper_fence)] print(f"Q1={Q1:,.0f} Q3={Q3:,.0f} IQR={IQR:,.0f}") print(f"Fences: [{lower_fence:,.0f}, {upper_fence:,.0f}]") print(f"Outliers: {outliers.tolist()}") print(f"Clean data: {len(clean)} of {len(sales)} rows")
Output
Q1=46,500 Q3=57,750 IQR=11,250 Fences: [29,625, 74,625] Outliers: [1200000] Clean data: 7 of 8 rows

🧹 Removing Outliers

Unit 6 — Real-world data is messy. Learn to find and fix every type of data quality problem.

Strategies for Handling Outliers

python
import pandas as pd, numpy as np df = pd.DataFrame({ "Sales": [45000,52000,48000,1500000,58000,31000,55000,-5000], "Region": ["Cairo","Alex","Cairo","Cairo","Giza","Alex","Giza","Cairo"], }) # Strategy 1: Remove (drop the rows) Q1, Q3 = df["Sales"].quantile([0.25, 0.75]) IQR = Q3 - Q1 mask = (df["Sales"] >= Q1-1.5*IQR) & (df["Sales"] <= Q3+1.5*IQR) df_removed = df[mask] print(f"After removal: {len(df_removed)} rows") # Strategy 2: Cap (Winsorize) — replace with fence values lo, hi = Q1 - 1.5*IQR, Q3 + 1.5*IQR df_capped = df.copy() df_capped["Sales"] = df_capped["Sales"].clip(lower=lo, upper=hi) print("After capping:") print(df_capped["Sales"].describe().round(0)) # Strategy 3: Transform (log scale reduces impact) df_valid = df[df["Sales"] > 0].copy() df_valid["Sales_Log"] = np.log(df_valid["Sales"]) print("\nLog-transformed (no outlier effect):", df_valid["Sales_Log"].std().round(2))
💡 Tip: Capping (Winsorizing) is often better than removal — you keep all rows and reduce outlier influence without losing data.

What is Data Validation?

Validation checks that data meets expected rules and constraints — like age being 0-120, email having @, or sales being positive.

python
import pandas as pd, numpy as np df = pd.DataFrame({ "Age": [25, -5, 150, 30, 28], "Email": ["a@b.com","notanemail","c@d.io","","e@f.net"], "Salary": [9500, 8000, 12000, -1000, 0], "Score": [750, 300, 850, 1200, 600], }) # Define validation rules def validate(df): issues = [] # Age: must be 18-100 bad_age = df[(df["Age"] < 18) | (df["Age"] > 100)] if len(bad_age): issues.append(f"Invalid age: {len(bad_age)} rows") # Email: must contain @ bad_email = df[~df["Email"].str.contains("@", na=False)] if len(bad_email): issues.append(f"Invalid email: {len(bad_email)} rows") # Salary: must be positive bad_salary = df[df["Salary"] <= 0] if len(bad_salary): issues.append(f"Non-positive salary: {len(bad_salary)} rows") # Score: must be 300-850 bad_score = df[(df["Score"] < 300) | (df["Score"] > 850)] if len(bad_score): issues.append(f"Invalid score: {len(bad_score)} rows") return issues for issue in validate(df): print(f" ⚠️ {issue}")
Output
⚠️ Invalid age: 2 rows ⚠️ Invalid email: 2 rows ⚠️ Non-positive salary: 2 rows ⚠️ Invalid score: 1 rows

Why Encode Categories?

Machine learning models need numbers, not text. Encoding converts categorical columns to numeric representations.

python
import pandas as pd df = pd.DataFrame({ "City": ["Cairo","Alex","Cairo","Giza","Alex","Cairo"], "Level": ["Junior","Senior","Mid","Senior","Junior","Mid"], "Score": [72, 95, 84, 91, 68, 79], }) # 1. Label Encoding — assign integers from sklearn.preprocessing import LabelEncoder le = LabelEncoder() df["City_Label"] = le.fit_transform(df["City"]) print("Label encoded:", dict(zip(le.classes_, range(len(le.classes_))))) # 2. One-Hot Encoding — separate column per category df_ohe = pd.get_dummies(df["City"], prefix="City") print(df_ohe) # 3. Ordinal Encoding — maintain order level_order = {"Junior":1, "Mid":2, "Senior":3} df["Level_Ord"] = df["Level"].map(level_order) print(df[["Level","Level_Ord"]])
Output
Label encoded: {'Alex': 0, 'Cairo': 1, 'Giza': 2} City_Alex City_Cairo City_Giza 0 0 1 0 1 1 0 0 ... Level Level_Ord 0 Junior 1 1 Senior 3 2 Mid 2
💡 Tip: Use ordinal encoding for ordered categories (Junior < Mid < Senior). Use one-hot for unordered categories (City, Product). Avoid label encoding for unordered — it implies Cairo(1) < Giza(2) which is wrong.

Quantile-Based Binning

python
import pandas as pd, numpy as np np.random.seed(42) sales = pd.Series(np.random.normal(50000,15000,100).round(0)) # qcut — equal-frequency bins (same number in each bin) quartiles = pd.qcut(sales, q=4, labels=["Q1 Low","Q2","Q3","Q4 High"]) print("Distribution:") print(quartiles.value_counts().sort_index()) # Custom bins for business logic performance = pd.cut(sales, bins = [-np.inf, 30000, 50000, 70000, np.inf], labels = ["Poor","Average","Good","Excellent"] ) print("\nPerformance:") print(performance.value_counts())
💡 Tip: Use pd.cut() for fixed boundaries (e.g., age groups). Use pd.qcut() for equal-sized groups (percentile-based).

🧹 Full Cleaning Pipeline

The Complete Pipeline

python
import pandas as pd, numpy as np def clean_sales_data(df): """ Complete data cleaning pipeline. Returns cleaned DataFrame and a report dict. """ report = {"original_rows": len(df), "steps": []} # Step 1: Standardize column names df.columns = (df.columns.str.strip() .str.lower() .str.replace(r"[\s]+","_",regex=True)) report["steps"].append("Column names standardized") # Step 2: Remove exact duplicates before = len(df) df = df.drop_duplicates() report["steps"].append(f"Removed {before-len(df)} duplicate rows") # Step 3: Fix data types if "date" in df.columns: df["date"] = pd.to_datetime(df["date"], errors="coerce") if "sales" in df.columns: df["sales"] = pd.to_numeric(df["sales"], errors="coerce") # Step 4: Handle missing values num_cols = df.select_dtypes(include="number").columns str_cols = df.select_dtypes(include="object").columns df[num_cols] = df[num_cols].fillna(df[num_cols].median()) df[str_cols] = df[str_cols].fillna("Unknown") # Step 5: Remove outliers (IQR) if "sales" in df.columns: Q1,Q3 = df["sales"].quantile([0.25,0.75]) IQR = Q3-Q1 before = len(df) df = df[(df["sales"] >= Q1-1.5*IQR) & (df["sales"] <= Q3+1.5*IQR)] report["steps"].append(f"Removed {before-len(df)} outliers") # Step 6: Clean strings for col in str_cols: df[col] = df[col].str.strip().str.title() report["final_rows"] = len(df) return df, report
💡 Tip: Always save original data before cleaning: df_original = df.copy(). Document every cleaning step for reproducibility.
